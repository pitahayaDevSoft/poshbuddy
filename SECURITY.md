# Security Policy

## Supported Versions

We actively support and patch the latest release of PoshBuddy. If you find a security issue, please ensure you are testing on the latest release before reporting.

| Version | Supported |
| ------- | --------- |
| >= 0.4.x |  ✅ Yes   |
| < 0.4.x |  ❌ No    |

## Reporting a Vulnerability

If you discover a security vulnerability within PoshBuddy, please do **NOT** open a public issue. Instead, report it responsibly by contacting the maintainer directly:

- **Contact:** Julio César Martínez
- **Email:** [julioglez.93@gmail.com](mailto:julioglez.93@gmail.com)

Please include the following information in your report:
1. A detailed description of the vulnerability.
2. Step-by-step instructions or a Proof of Concept (PoC) to reproduce the issue.
3. The potential impact of the vulnerability.

### Our Commitment

We take security seriously and commit to the following response timeline:
- **Acknowledgement:** We will acknowledge receipt of your vulnerability report within **48 hours**.
- **Assessment:** We will assess and validate the vulnerability within **7 days**.
- **Remediation:** We will work on a patch and release a secure update within **30 days**, keeping you updated throughout the process.
